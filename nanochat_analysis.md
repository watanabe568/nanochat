# nanochat リポジトリ主要フォルダ調査

## 概要

nanochat は、大規模言語モデル（LLM）の学習から推論までを実装した教育用プロジェクトである。本資料では、リポジトリ内の主要フォルダである **nanochat**、**scripts**、**tasks** の役割と、それぞれに含まれるファイルの役割についてまとめる。

---

# 1. nanochat フォルダ

## 役割

nanochat フォルダは、LLM の中核機能を実装するフォルダである。

モデルの構造定義、データ処理、学習済みモデルの管理、推論処理など、システムのコア機能が格納されている。

## 主なファイル

| ファイル名                 | 役割                        |
| --------------------- | ------------------------- |
| gpt.py                | GPTモデル（Transformer）の構造を定義 |
| engine.py             | 推論エンジンとして動作し、応答生成を行う      |
| dataset.py            | 学習データの取得・前処理を行う           |
| dataloader.py         | 学習データをミニバッチ形式で読み込む        |
| core_eval.py          | モデルの評価処理を実行する             |
| checkpoint_manager.py | 学習済みモデルの保存・読み込みを行う        |
| ui.html               | Webチャット画面のUI              |
| rustbpe/              | BPEトークナイザーの学習およびトークン化処理   |

## まとめ

nanochat フォルダは以下の機能を担当している。

* モデル構造の定義
* 学習データ処理
* 推論処理
* 評価処理
* トークナイザー管理

つまり、nanochat 全体の「エンジン部分」を構成するフォルダである。

---

# 2. scripts フォルダ

## 役割

scripts フォルダには、学習・評価・チャット実行を行うための実行用スクリプトが格納されている。

nanochat フォルダ内の機能を呼び出して処理を実行する。

## 主なファイル

| ファイル名         | 役割                     |
| ------------- | ---------------------- |
| base_train.py | 事前学習（Pretraining）を実行   |
| mid_train.py  | 中間学習（Mid-training）を実行  |
| chat_sft.py   | SFT（教師ありファインチューニング）を実行 |
| chat_eval.py  | 学習済みモデルの評価を行う          |
| chat_web.py   | Webチャットサーバを起動する        |
| chat_cli.py   | コマンドラインでチャットを実行する      |
| chat_rl.py    | 強化学習（RLHF系）の学習を行う      |

## 学習の流れ

1. base_train.py

   * 大量のテキストデータを用いて事前学習

2. mid_train.py

   * 特定タスク向けに追加学習

3. chat_sft.py

   * 対話データによる教師あり学習

4. chat_eval.py

   * モデル性能の評価

5. chat_web.py

   * Webチャットとして公開

## まとめ

scripts フォルダは、学習・評価・実行を行うための操作用プログラム群である。

ユーザーが直接実行するファイルが主に格納されている。

---

# 3. tasks フォルダ

## 役割

tasks フォルダは、モデルの学習や評価で利用するベンチマークタスクを管理するフォルダである。

さまざまな問題セットを通して、モデル性能を測定できる。

## 主なファイル

| ファイル名        | 内容              |
| ------------ | --------------- |
| common.py    | タスク共通クラス        |
| arc.py       | ARC（科学問題ベンチマーク） |
| mmlu.py      | MMLU（総合知識評価）    |
| gsm8k.py     | 数学文章題評価         |
| humaneval.py | コード生成能力評価       |

## 各タスクの概要

### ARC

科学分野の選択問題を解くベンチマーク。

知識問題や論理的推論能力を評価する。

### MMLU

57分野にわたる総合学力テスト。

以下のような分野が含まれる。

* 数学
* 物理
* 医学
* 法律
* コンピュータサイエンス

### GSM8K

小中学生レベルの文章題を解くベンチマーク。

計算能力や論理的思考力を評価する。

### HumanEval

プログラム生成能力を評価するベンチマーク。

与えられた関数仕様に対して正しいコードを生成できるかを測定する。

### common.py

すべてのタスクの共通機能を提供する。

主な機能は以下の通り。

* データ取得
* データ管理
* 評価処理
* 共通インターフェース提供

## まとめ

tasks フォルダは、モデルの学習および評価に利用する問題集やベンチマークを管理するフォルダである。

---

# 全体構成

| フォルダ     | 役割                  |
| -------- | ------------------- |
| nanochat | LLM本体（モデル・推論・データ処理） |
| scripts  | 学習・評価・実行スクリプト       |
| tasks    | 評価用ベンチマーク・問題集       |

## システム全体の流れ

```text
tasks
  ↓
scripts
  ↓
nanochat
  ↓
学習済みモデル生成
  ↓
chat_web.py
  ↓
ブラウザでチャット
```

## 結論

nanochat リポジトリは以下の3つのフォルダを中心に構成されている。

* nanochat：LLM本体を実装するエンジン部分
* scripts：学習や評価を実行する操作部分
* tasks：学習および評価用データセット管理部分

それぞれが連携することで、モデルの学習から推論、評価までを一貫して実行できる構成となっている。
