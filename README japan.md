Nanochat は、大規模言語モデル（LLM）をトレーニングするための、最小限でハッカブルなフルスタックの実験用ハーネスです。生のテキストのトークナイズから、事前学習、教師あり微調整（SFT）、強化学習（RL）、そしてチャット UI を介した最終的なデプロイまで、LLM のライフサイクル全体をカバーするように設計されており、そのすべてが単一の GPU ノードに最適化された単一のリポジトリ内に収められています [README.md:1-6]。

このプロジェクトの主要なベンチマークは GPT-2 Speedrun です。これは、DCLM CORE スコアで測定される GPT-2 グレードの能力を持つモデルを、最短の実時間でトレーニングすることを目的としています。現在の記録では、2019 年のオリジナルの GPT-2 を凌駕するモデルを、8xH100 ノードを使用して約 1.65 から 2 時間、50 ドル未満のコストでトレーニングできることが示されています [README.md:10-24]。

設計思想：単一の深度ダイヤル (Single-Depth Dial)
Nanochat は「単一ダイヤル」の設計思想に基づいて構築されています。ユーザーが数十のハイパーパラメータを手動で調整する代わりに、--depth パラメータ（トランスフォーマーのレイヤー数）を 1 つ指定するだけで、システムがスタックの残りの部分に最適な構成を自動的に導き出します [README.md:6-7]。

導出されるハイパーパラメータ	導出ロジック
モデルの幅 (Width)	アーキテクチャのバランスを維持するために、深度に基づいてスケーリングされます。
学習率 (Learning Rate)	スケーリング則を介して調整され、多くの場合 sqrt(batch_ratio) によってスケーリングされます [scaling_laws.sh:1-20]。
トレーニング期間 (Training Horizon)	target-param-data-ratio（デフォルトは約 10.5）に非埋め込みパラメータ数を乗じて算出されます [LEADERBOARD.md:36-37]。
オプティマイザ	内部表現には Muon を使用し、スカラーと埋め込み（Embedding）には AdamW を使用します [nanochat/optim.py:1-50]。
Nanochat における LLM ライフサイクル
このコードベースは完全なパイプラインを実装しており、研究者は「ゼロ」の状態から機能的なチャットアシスタントを作成できます：

トークナイゼーション: 数字や空白に特定のパターンを用いた BPE トークナイザーのトレーニング [nanochat/tokenizer.py:1-30]。
事前学習: ClimbMix のような高品質な混合データセットを用いてベースモデルをスケーリング [dev/LOG.md:107-113]。
SFT: 会話パッキングとアシスタントのみの損失マスキングを用いたインストラクションチューニング [scripts/chat_sft.py:1-40]。
RL: GRPO スタイルの強化学習を用いたモデルの整列（アライメント） [scripts/chat_rl.py:1-20]。
推論: KV キャッシュとツール利用機能を備えた高性能エンジン [nanochat/engine.py:1-50]。
